# SpaDES.project repo conventions

## NEWS.md

Keep entries **very concise — generally one line per entry**. Group under
existing section headers (`## New features`, `## Enhancements`, `## Bug
fixes`, `## Removed`). Lead with the function / object name in
backticks, then the change. Skip implementation details that belong in
the commit message or roxygen docs; NEWS.md is for users skimming
"what's new" between versions.
